# Multi-Screenshot Narrated Bundle — Smoke Test Checklist

Run through these manually on a physical Apple Silicon Mac before claiming Spec 1 implementation complete. Mark each item ✓ as you confirm it passes.

## Prerequisites
- [ ] `pnpm tauri dev` running cleanly (no Rust compile errors)
- [ ] `pnpm dev:proxy` running in another shell (vp-edge mock)
- [ ] Mic permission granted to the dev binary (System Settings → Privacy & Security → Microphone → VisionPipe)
- [ ] Screen Recording permission granted to the dev binary
- [ ] Accessibility permission granted to the dev binary
- [ ] `DEEPGRAM_API_KEY` is set in `.env.local` for end-to-end transcription tests

## First-launch onboarding (Onboarding card)

- [ ] Fresh launch (or revoke a permission first) → onboarding card appears at app open
- [ ] All 5 permission rows render (Screen Recording, System Events, Accessibility, Microphone, Speech Recognition)
- [ ] Click "Open System Settings" on a missing permission → System Settings opens to the relevant pane
- [ ] After granting in System Settings and tabbing back, the green checkmark appears within ~2 seconds (auto-poll)
- [ ] When all 5 are green, "Got it" button appears; clicking it dismisses the onboarding card

> **Known bug:** Speech Recognition's "Open System Settings" button may not deep-link correctly on macOS 14+. The `Privacy_SpeechRecognition` URL scheme has changed; the user can grant manually via the Privacy panel.

## Happy path: capture + narrate

- [ ] Press `Cmd+Shift+C` (or whatever you set in Settings) → region-select overlay appears with crosshair + "Drag a region · Enter for fullscreen · Esc to cancel" pill
- [ ] Drag a region → session window opens, sized to ~70% × 85% of monitor, centered (NOT fullscreen)
- [ ] One screenshot card renders with the captured image as a thumbnail
- [ ] Mic indicator in header shows "Recording · Live" (red dot, "Live" label means Deepgram WebSocket connected)
- [ ] Talk for 5–10 seconds — finalized transcript text appears in the active card's narration textarea
- [ ] Click "+ Take next screenshot" → window hides → drag region → window reappears with second card
- [ ] First card's `audioOffset.end` is set in `transcript.json`; second card's `audioOffset.start` matches (inspect via `cat ~/Pictures/VisionPipe/session-*/transcript.json | jq .screenshots`)
- [ ] Edit caption on card #1 → wait 500ms → `transcript.json` updates with the new caption (debounced auto-save)
- [ ] Click 🎙 on card #1 → modal opens → talk 5s → click Stop
- [ ] `<canonicalName>-rerecord.webm` exists in session folder; `transcript.json` shows `reRecordedAudio` set on that screenshot
- [ ] Click "Detach transcript" in header → split view (cards left, transcript right). Click "Attach transcript" → back to interleaved
- [ ] Quit app, re-launch, take a new capture — the SplitView toggle persists (your last-used view is the default)
- [ ] Click "📋 Copy & Send" → paste into a text editor → verify markdown structure has `# VisionPipe session`, `## Screenshot N`, `**Caption:**`, `**Context:**`, `**Narration:**`, image references with absolute paths, closing narration, `*Generated by VisionPipe*`
- [ ] Click thumbnail on card → lightbox opens at full resolution
- [ ] Press Esc → lightbox closes
- [ ] Press `Cmd+Enter` (or your configured Copy & Send hotkey) → markdown re-copies to clipboard
- [ ] Press `Cmd+T` → view toggles
- [ ] Press `Cmd+Shift+R` → re-record modal opens for the most recent screenshot

## Offline path

- [ ] With session active, stop `pnpm dev:proxy` (kill the process)
- [ ] Within ~5 seconds, mic indicator switches to "Reconnecting…", then to "Local-only" within ~3 more seconds
- [ ] Take a 3rd screenshot — its narration area shows "(offline — audio recorded locally; no transcript)" placeholder
- [ ] `audio-master.webm` is still recording (file size grows)
- [ ] Restart `pnpm dev:proxy` → take a 4th capture; this one streams real-time transcripts again
- [ ] Click "Copy & Send" — markdown for screenshot #3 contains the offline placeholder; #4 contains a real transcript

## Crash recovery

- [ ] Take 2 screenshots in a session
- [ ] Force-quit the app (`pkill -KILL VisionPipe` or `pkill -KILL -f tauri`)
- [ ] Re-launch
- [ ] Open `~/Pictures/VisionPipe/` in Finder — the session folder is there with both screenshots, `transcript.json`, partial `audio-master.webm` (likely missing the last 1s chunk)
- [ ] Take a new capture in the relaunched app — a NEW session folder is created (no auto-resume in v0.2)

## Long session

- [ ] Take 20 screenshots in a single session
- [ ] UI stays responsive
- [ ] `transcript.json` updates after each capture
- [ ] Click "Copy & Send" — markdown is well-formed
- [ ] Paste into Claude Code session → verify all 20 image references resolve via `Read` tool

## Hotkey rebinding

- [ ] Open Settings (overflow menu → 3) → change "Take next screenshot" to `Cmd+Shift+P`
- [ ] Quit and re-launch app
- [ ] Pressing `Cmd+Shift+P` triggers capture; `Cmd+Shift+C` no longer does
- [ ] Try to bind to `Cmd+Q` in Settings → conflict warning "Reserved by macOS (Quit)" appears
- [ ] Click "Reset all to defaults" → restores `Cmd+Shift+C`

## Permission denial

- [ ] Revoke mic permission for the dev binary
- [ ] Take a screenshot — session opens, mic indicator shows "Paused" (disabled button), banner not shown but capture still works
- [ ] Manually type narration in the textareas → markdown output omits the Narration block visually only when `transcriptSegment` is empty AND `offline` is false (current behavior renders the narration with whatever's typed)

## Window resize

- [ ] After first capture, the session window is ~70% × 85% of monitor (verify by checking it doesn't fill the entire screen)
- [ ] Window stays in the same position when taking subsequent screenshots (selection overlay covers fullscreen, but the session window snaps back)
- [ ] On a 4K monitor, the window caps at 1600×1000 logical (so it doesn't dominate)
- [ ] On a 13" MacBook Air (1280×800), the window caps at 800×600 minimum (footer always visible)

## Verify on actual Apple Silicon

- [ ] Test on M1 (or M2/M3/M4) Mac
- [ ] Capture latency feels instant (< 100ms perceptual)
- [ ] Audio recording stays glitch-free for 5+ minutes
- [ ] No memory leaks during a 10-minute session (check Activity Monitor)

## Verify the spec's golden output

- [ ] Run `pnpm test src/lib/__tests__/markdown-renderer.test.ts` — all 4 golden tests pass
- [ ] Inspect a real session's `transcript.md` against the golden fixture in `src/lib/__tests__/__fixtures__/session-2-screenshots.expected.md` — structure matches (the field values differ since they're real captures)
